Flow boundary and barriers
For all 2D flow:
0 - outflow not allowed except in outflow points in outlet.map. Water can pool at the edges if the DEM is not perfect.
1: outflow to all sides possible of flood and overland flow. Outlets always allow outflow. (default)
2: a user defined map (flowboundary.map) allows 2D outflow where the value is 1, and a closed boundary where the value = 0. A map filled with 1 is the same as unchecked, free outflow. NOTE: if you are modelling a boundary with the sea (island), do not close the boundary.

 

Buffers

This uses a map that is added to the DEM, works only for 2D flow. Positive values (in buffers.map) are gridcells added to the DEM (in meters), negartive values are values that are subtracted from the DEM.  

 

Flow Barriers between cells
Barriers up to a given height (m) to disconnect flow between cells (only for 2D flow). The barriers are assumed to be on the cell boundary and have no physical thickness, only a height. Cells can be disconnected in the 4 directions NESW. This needs flowbarrier.map with codes 0 (no barrier), and 1, 2, 3 ... n. The numbers correspond to the textfile flowbarriers.txt where the height of the barriers are given.

 

Coupling kinematic and diffusive overland flow to dynamic flow for flooding

  • Manning n coefficient:  Overland flow continues in the flooded area but Manning's n is linked to the flood water depth (hf in m) so the runoff slows down: n = n * exp(cf * hf)
  • Flood mixture coefficient: flood level is increased with incoming overlandflow where the two overlap. The overland flow height (hof) is decreased with: dH=WH*(1-exp(-cp*hof/hf))
    and the flood level hf is increased with the same amount. The function is bounded between 0 and 1.

2D Diffusive overland flow parameters

  • Min timestep for 2D flow is default at 0.1 second. The 2D flow is non-iterative. The minimjum timestep helps to maintain a stable flow. Make this smaller when a unstable pattern appears (e.g. checkboard pasttern).
  • The courant condition helps to keep the flow stable (smaller is more stability). Default value 0.2.
  • Concentrated flow factor: this empirical weight is applied to te direction with the largest flow to emphasize that direction. This effect may not be very apparent because it can also be limited in subsequent timesteps depending on the DEM.

2D dynamic wave parameters
A MUSCL (Monotonic Upstream-Centered Scheme for Conservation Laws), uses reconstructed states, derived from cell-averaged states obtained from the previous time-step. For each cell, slope limited, reconstructed left and right states are obtained and used to calculate fluxes at the cell boundaries (edges).

  • Cell centred solution (0): Water pressure and velocities on cell boundaries based on a 1st order taylor solution, only using information in the cell itself. Faster but less accurate.
    MUSCL solution (1):     Water pressure and velocities on cell boundaries based on a 2st order taylor solution, using information nin the cell and neighbours in X and Y direction. Slower and more accurate.
  • Courant factor: Determines minimum timstep, lower means smaller timesteps in the iteration are used. (def. 0.2, range 0.01-1.0).
  • Minimum timestep: User defined minimum timestep for dynamic wave solution. In case of instability lower this value (def. 0.1 sec).
  • Spatially variable timestep: use a local smallest timestep (checked) for the numerical solution in a 5x5 window instead of a global smallest timestep (unchecked).
  • Time average velocity: Velocity of cells is a weighted average between two iteration steps (def on).
