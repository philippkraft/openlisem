Overland flow

  • 1D uses a kinematic wave on a predefined continues network that connects all cells to the outlet of the area (ldd.map). The network is user defined and is an input, the DEM is not used. The kinematic wave uses a finite difference iteration. If there is no channel activated, there is only runoff and no flood. If there is a channel and flooding, this is always simulated with a dynamic wave.
  • 2D dynamic flow uses the DEM to determine where water flows. The water is distributed over the cells downstream, using a full dynamic wave for shallow flow. 'Shallow' means that an average velocity is used for the depth. The 'flow' tab gives several numerical options. A small adaptive timestep (non-iterative) is used to ensure stability.

NOTE: if a channel is ativated, all overflow from the channel on land is always simulated using a 2D dynamic wave.

NOTE: channel flow is always 1D kinematic wave over a channel network.

Flood threshold

Independent of the numerical flow simulation, flood is regarded from a practical stakeholder point of view! All overland flow below a certain depth threshold is regarded as runoff, above this threshold it is regarded as flood. The threshold is user defined (default 5 cm). This threshold also affects flood timing and duration and flood statistics.

Sediment processes

  • Switch erosion processes on or off. If switched on, sediment dynamics are simulated for for all flows. Only suspended matter is simulated (no bedload) using the Govers transport capacity equation.
  • Advanced sediment transporty allows you to simulate suspended and bedload transport, with different transport capacity equations, mulitple grainsize classes, and sediment material depth.


Channels/Rivers

The channel system can be switched off entirely in which case only overland flow is considered (for research purposes). Channels are 1D networks using a kinematic wave for flow.

  • Channel infiltration: assumed to be saturated infiltration, the flux equals ChannelKsat (mm/h). If not checked the channel is assumed impermeable. Mutually exclusive with baseflow.
  • Channel baseflow: the baseflow discharge at the outlet(s) is given in m3/s in the map baseflow.map. This discharge is iterated over the entire channel, so that the kinematic wave gives the stated baseflow at the outlet. Mutually exclusive with channel infiltration.


Infrastructure

Switch on or off the effect of buildings and infrastructure:

  • Buildings have interception storage, are impermeable, obstruct flow and are non-erodable. They can have storage drums for rainwater (interception from roof goes into the drum). Buildings can be smaller than the girdcell in which case the characteristics of the soil and vegegation are used.
  • Roads have no interception, are impermebale, smooth, have no detachment of sediment but can have depsition. Roads can be smaller than the girdcell in which case the characteristics of the soil and vegegation are used.
  • Hard Surfaces (parking lots, courtyards, airstrips), have no interception, are impermebale, smooth, have no detachment of sediment but can have depsition. They behave the same as roads but are a separate class because they are often in a different land use class.
  • Storm drains (tile drains) are networks of circular subsurface pipes with regular inlets. A kinematic wave transports water along this system. The maximum flow is determined by the diameter, gradient and Manning's n. Can be used for urban drains or agricultural tile drains.

Parallel processing

All CPUs have more than 1 core that can be activated simultatneously for calculations. If the value is set at 0, all cores present are used. The flow domain is divided in areas that are calculated in parallel with a narrow band to connect these areas when they are done. This is usually faster (20% and 50%) for large areas but may also lead to errors. It is NOT faster for small areas. For small areas there is hardly any difference. Defualt = 1 (1 core used). Note that writing results to disk and screen is always done in a separate core.
